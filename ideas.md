我需要完成两个大团队的整合，需要吸收很多新成员进来，我需要对很多人进行1:1的交流。
交流时我需要只言片语地记录一些人才的要点。同时我需要对所有录入过信息的团队成员都整理成人才卡。
因此我需要一个网站，来展示“人才卡”和“信息录入”以及“信息查询”几个核心功能。注意信息查询可能是模糊的描述。
这个网页需要能支持iphone 12 pro max的访问。也需要可以支持pc端的访问。
从实现上来说，我们采用LLM来将录入的人才信息分门别类地整理到人才卡上，每个人才卡的维度应该对齐。
信息录入和信息查询都使用LLM来辅助完成。我们采用Gemini的API。
以下是实现上的建议：
1. 建立一个docker/文件夹，构建一个Dockerfile，使得网站和全部运行环境都基于容器来做，可以通过docker-compose来管理容器
2. 建立一个config/文件夹，在里面有个示例的config yaml文件，也有一个真实使用的，在里头记录诸如gemini api key等敏感信息
3. 网站需要使用密码登录，**密码认证 + 渐进式封禁 + 自动解封**

| 措施 | 说明 |
| --- | --- |
| 强密码 | 足够长的随机密码，暴力破解不现实 |
| 渐进式封禁 | 5次失败封30分钟，10次封2小时，20次封24小时 |
| 自动解封 | 封禁到期自动恢复，不会把自己锁死 |
| rate limiting | 每IP每分钟最多5次登录请求 |

这样既能有效阻止暴力破解（攻击者每换一个IP也只能试几次，效率极低），又不会误伤自己。
密码从config yaml读取
4. 如果本地没有设置密码，显示一个菜单页，暗示没有设置哨兵，同时在后端的日志警告具体的问题和修正方案
5. 信息录入页面支持一个对话框，输入针对某个候选人的信息，支持一个下拉菜单选中某个候选人，选中过程支持输入匹配搜索以减少选择时间。信息录入页面也需要支持上传一个人才的简历PDF，需要找到办法识别这个PDF，并提取信息更新到人才卡上
6. 输入的信息由LLM整理，更新到人才卡上
7. 人才卡页面支持按tag分类展示不同人才卡，点开可以展示详情，可以支持搜索
8. 人才卡支持导出为PDF，生成一个完整对人才的描述
9. 在手机的人才卡页面上有个快捷搜索的设计，支持快速搜索某个人，支持用拼音模糊搜索
10. 所有数据保存在本地的数据库，可以使用sqlite，并每天定期上传保存到腾讯云的cos上，cos的credential记录在config的yaml里 
11. 网站通过https访问，后续我会绑定我的域名
12. web的启动脚本支持start_web.sh和restart_web.sh，如果反复调用start_web.sh也会删掉之前的进程再重启一个新的
13. 在根目录创建一个README.md，详细解释用法和要点，并提示如何通过cloudflare绑定域名以及映射端口
